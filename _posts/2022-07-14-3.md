동업할때 다른 사람이 바꾸지 말았으면 하는 소스가 있기 마련입니다. 예를 들면 공통 컴포넌트, 프로젝트의 환경설정 등이 있겠죠.

그러나 자신이 원하는 기능구현을 위해 공통소스를 함부로 건드리는 사람이 한 명씩 꼭 있습니다. (저도 회사에서 종종 경험합니다.)

또는 디버깅을 위해 잠시 바꿔놓은 소스를 커밋하기 전에 되돌리는 것을 까먹고 그대로 커밋하는 경우도 있습니다.

이를 방지하기 위한 방법으로 `npm`의 라이브러리 `husky`를 소개합니다.

## Git Hook

`husky` 는 git hook을 제어하기 위한 라이브러리인데, git Hooks 는 git 과 관련한 어떤 이벤트가 발생했을 때 특정 스크립트를 실행할 수 있도록 하는 기능입니다.

실행하고자 하는 스크립트를 클라이언트에서 실행하는지 서버에서 실행하는지에 따라 클라이언트 훅, 서버 훅으로 나뉩니다.

우리가 사용할 것은 클라이언트 훅 중에서도 `pre-commit` 과 `pre-push` 입니다.

`pre-commit` 과 `pre-push` 는 각각 커밋과 푸시 전에 실행되며, 상황에 따라 커밋과 푸시를 중단시킬 수 있습니다.

이를 이용해 특정 경로 안에 변화가 감지될 경우 커밋과 푸시를 방지하도록 해보겠습니다.

<br>

`npm install -D husky@4` 명령어로 `husky` 4 버전을 설치하고 `package.json`에 다음과 같이 `husky` 속성을 추가합니다.

```json
  "devDependencies": {
    ...
    "husky": "^4.3.8",
    ...
  },
  "husky": {
    "hooks": {
      "pre-commit": "node gitHook.js pre-commit",
      "pre-push": "node gitHook.js pre-push"
    }
  },
```

위에 작성한 대로라면 `pre-commit` 시에 `node` 환경으로 `gitHook.js`를 `'pre-commit'` 이라는 인자값으로 실행하며

`pre-push` 시에는 `gitHook.js`를 `'pre-push'` 라는 인자값으로 실행합니다.

이제 파일 변화를 감지하는 스크립트를 `gitHook.js`에 작성할껀데 이를 위해 몇가지 `git` 명령어를 살펴보도록 합니다.

## git diff

`git diff` 명령어는 git을 통해 변화가 감지된 부분을 출력합니다.

일반적으로는 수정된 코드 전체가 출력되지만 우리는 파일 명에만 관심있기 때문에 `--name-only` 옵션을 사용합니다.

<br>

원격 저장소에 올라가기까지 아래의 과정을 거칩니다.

<br>

> **작업공간** &rarr; `git add` &rarr; **스테이지 영역** &rarr; `git commit` &rarr; **로컬 저장소** &rarr; `git push` &rarr; **원격 저장소**

<br>

우리에게 필요한건 커밋하기 전 스테이지 영역에서 일어나는 파일 변화와 푸시하기 전 로컬 저장소에서 원격 저장소 간의 파일 변화입니다.

- `pre-commit` 에서는 스테이지 영역에 올라가있는 변화를 인식하는 `git diff --name-only --cached`
- `pre-push` 에서는 로컬 저장소와 원격 저장소 간의 변화를 인식하는 `git diff --name-only HEAD origin/[브랜치명]`

<br>

`pre-push` 에서는 브랜치명이 필요한데 이는 작업 환경마다 항상 다를 수 있죠.

그리하여 현재 브랜치명을 출력하는 명령어인 `git rev-parse --abbrev-ref HEAD` 의 결과값을 브랜치명으로 넣어주도록 하겠습니다.

<br>

저는 아래와 같이 `gitHook.js` 를 작성했습니다.

```jsx
const { exec } = require("child_process");

// 커밋을 방지할 디렉토리
const PROTECTED_DIR = ["src/common", "environment"];

// 경로 형식을 '/' 로 통일
const dirFormat = (dir) => {
  return dir.replace(/^[\/+|\\+]/, "").replace(/\/+|\\+/g, "/");
};

// pre-commit을 인자로 받았을때 실행
if (process.argv.slice(-1)[0] === "pre-commit") {
  exec("git diff --name-only --cached", (error, stdout, stderr) => {
    if (error) throw error;
    const protectedFiles = [];
    stdout.split(/\s+/).forEach((fileName) => {
      if (PROTECTED_DIR.some((dir) => dirFormat(fileName).startsWith(dir)))
        protectedFiles.push(fileName);
    });

    if (protectedFiles.length) {
      console.error(
        "\x1b[31m%s\x1b[0m",
        "보호된 디렉토리에 수정이 감지되었습니다:\n\t" +
          protectedFiles.join("\n\t")
      );
      process.exit(1);
    }
  });
}

// pre-push를 인자로 받았을때 실행
if (process.argv.slice(-1)[0] === "pre-push") {
  exec("git rev-parse --abbrev-ref HEAD", (error, stdout, stderr) => {
    if (error) throw error;

    exec(
      `git diff --name-only HEAD origin/${stdout}`,
      (error, stdout, stderr) => {
        if (error) throw error;
        const protectedFiles = [];

        stdout.split(/\s+/).forEach((fileName) => {
          if (PROTECTED_DIR.some((dir) => dirFormat(fileName).startsWith(dir)))
            protectedFiles.push(fileName);
        });

        if (protectedFiles.length) {
          console.error(
            "\x1b[31m%s\x1b[0m",
            "보호된 디렉토리에 수정이 감지되었습니다:\n\t" +
              protectedFiles.join("\n\t")
          );
          process.exit(1);
        }
      }
    );
  });
}
```

간단히 요약하자면 `child_process` 모듈의 `exec` 메소드로 git 명령어를 실행하고 결과값을 처리하기 쉬운 형태로 가공합니다.

파일명이 `PROTECTED_DIR` 에 정의된 디렉토리 중에서 시작되는 경우 `process.exit(1)` 로 커밋 또는 푸시를 막습니다.

## `--no-verify` 옵션

특별한 상황에서 `PROTECTED_DIR` 에 포함된 영역을 수정해야 하는 경우가 있을 수 있습니다.

`--no-verify` 옵션을 사용하면 `pre-commit` 또는 `pre-push` 를 스킵하도록 할 수 있습니다.

그러나 일반적인 상황에서는 쓰지 않도록 하며 꼭 수정해야 하는 경우 논의를 거쳐서 하도록 할 수 있겠습니다.
